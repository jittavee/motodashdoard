# รายละเอียดการส่งข้อมูลของ ESP32 ผ่าน BLE

## โครงสร้างข้อมูล

ESP32 จะส่งข้อมูลเป็นชุด (set) โดยใน 1 ชุดประกอบด้วย **13 packets** (13 parameters) แต่ละ packet จะถูกส่งแบบแยกรายการและมีการหน่วงเวลา (delay) ระหว่างการส่ง **15 ms** ต่อ packet เมื่อส่งครบทั้ง 13 packets แล้ว จะวนกลับไปเริ่มส่ง packet แรกใหม่

### ตัวอย่างข้อมูลที่ส่ง

| Parameter | ตัวอย่างค่า |
|-----------|-------------|
| EcuModel | 0, 1, 2, 3 |
| ECU | Connected, No_response, Connecting... |
| TECHO | 15000 |
| SPEED | 255 |
| WATER | 150 |
| AIR.T | 100 |
| MAP | 200 |
| TPS | 100 |
| BATT | 29.0 |
| IGNITI | 75.0 |
| INJECT | 20.0 |
| AFR | 20.0 |
| S.TRIM | 100 |
| L.TRIM | 100 |
| IACV | 100 |

---

## รายละเอียด Packet

### โครงสร้าง

```
<ชื่อ parameter>=<ค่าของ parameter>
```

### รายละเอียด

| หัวข้อ | คำอธิบาย |
|--------|----------|
| **ชื่อ parameter** | เช่น TECHO, SPEED, BATT ใช้ระบุว่าค่าที่ส่งคือค่าของ parameter ใด |
| **เครื่องหมาย =** | ใช้เป็นตัวแบ่งระหว่างชื่อ parameter และค่าที่ส่ง |
| **ค่าของ parameter** | เป็นข้อมูลตัวเลข อาจมีทศนิยม (เช่น 29.0) |
| **ไม่มีอักขระเริ่มต้น/สิ้นสุด** | ไม่มีอักขระเริ่มต้น (Start) หรือสิ้นสุด (End) packet |
| **ความยาวข้อมูล** | ข้อมูลต่อ 1 packet ไม่เกิน **20 ตัวอักษร** |
| **รูปแบบการส่ง** | ข้อมูลถูกส่งในรูปแบบ **char string** โดยอ้างอิงรหัสตัวอักษรจาก **ASCII Table** |
| **การส่งผ่าน BLE** | Packet จะถูกเขียนเก็บไว้ใน Characteristic และส่งออกโดยใช้ **BLE Notify** |

## สรุปการทำงาน

1. ESP32 ส่งข้อมูลผ่าน BLE Notify
2. ส่งทีละ 1 packet ทุก 15 ms
3. ครบ 13 packets = 1 รอบข้อมูลสมบูรณ์
4. วนส่งซ้ำต่อเนื่อง
5. แอปพลิเคชันรับข้อมูลโดยการ subscribe BLE Characteristic และ parse ข้อมูลจาก string format
